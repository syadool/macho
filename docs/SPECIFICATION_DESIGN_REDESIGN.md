# デザイン刷新計画 — Apple Design 原則の適用

対象: MACHO(筋トレ記録Webアプリ / Next.js 15 + Tailwind v4)
種別: 刷新計画のみ(本ドキュメント時点で実装・修正は行わない)
参照: Apple *Designing Fluid Interfaces* (WWDC 2018) / *Principles of Great Design* ほか、apple-design スキル

## 1. 現状の評価

### 良い点(維持する)

- **明確なブランド**: ダーク基調 + ライム(`#d4ff00`)アクセント、Bebas Neue の数字表示。ジムで映える高コントラストな見た目はアプリの目的(達成感)に合っている。刷新でも配色・書体は原則維持する。
- **タップフィードバックの土台**: 主要ボタンに `active:scale` があり、`touch-action: manipulation` で300ms遅延も排除済み([globals.css:46](app/globals.css:46))。
- **一貫したカード言語**: `Card` / `Pill` / `PrimaryButton` が [ui.tsx](src/components/ui.tsx) に集約されており、トークン刷新の波及が効きやすい構造。
- **モバイル前提のレイアウト**: `PhoneShell` + `svh` + セーフエリア対応は直近のコミットで整備済み。

## 2. 刷新の方向性(デザイン原則)

Apple の8原則をこのアプリに翻訳する。

1. **Purpose** — このアプリの核は「ジムで片手・30秒で記録できる」こと。刷新はすべて記録フローの速さ・確実さに奉仕する。飾りのアニメーションは追加しない。
2. **Agency** — 削除・保存はすべて取り消せるようにする(Undo トースト)。確認ダイアログは「記録全体の削除」のみに限定。
3. **Familiarity** — iOS ネイティブアプリの操作語彙(シート、下スワイプで閉じる、ラバーバンド)に寄せる。ユーザーは iPhone でこのアプリを使う。
4. **Simplicity** — 記録画面は「よく使う道」を最短にする(前回コピー・サジェスト優先)。詳細調整は一段深く。
5. **Craft** — スペーシング・角丸・モーション値をトークン化し、全ての値に根拠を持たせる(§4, §5)。
6. **Delight** — 狙う感情は「達成感と勢い」。保存成功・ボリューム数値のカウントアップなど、達成の瞬間にだけ演出を集中させる。

## 3. モーションシステム(最重要の刷新)

### 3.1 ライブラリ導入

`motion`(旧 Framer Motion, 単体 `motion` パッケージ)を導入する。理由: スプリングが現在値・現在速度から再ターゲットでき(中断可能性)、`velocity` 受け渡しとジェスチャ(drag)を標準サポートするため。CSS transition はホバー等の非ジェスチャ系にのみ残す。

### 3.2 スプリング標準値(ハウススタイル)

Apple の damping / response をそのまま採用する。**固定 duration のアニメーションは原則禁止。**

| 用途 | bounce (damping相当) | duration (response相当) |
| --- | --- | --- |
| 標準UI(表示・切替・レイアウト) | `0` (damping 1.0) | `0.35` |
| シート・ドロワー | `0.15` (damping ~0.85) | `0.3` |
| フリック・ドラッグ解放後(勢いあり) | `0.2` (damping ~0.8) | `0.4` |

運用ルール:

- **バウンスはジェスチャに勢いがあった時だけ**。フェードインするメニューにオーバーシュートは付けない。
- **中断可能**: 進行中のアニメーション中も入力を受け付け、常に現在の表示値から再アニメーションする(motion のデフォルト挙動に乗る)。
- **ドラッグ解放時は指の速度をスプリングに渡す**(`velocity` オプション)。
- 2次元の移動は X/Y 独立スプリングに分解する。

### 3.3 ジェスチャ標準

- **タップ**: pointer-down で即ハイライト(scale 0.97)、up でコミット。現行の `active:scale` を維持しつつ、motion 化する箇所は `whileTap` に統一。
- **ドラッグ**: 開始判定 ~10px のヒステリシス。掴んだ位置のオフセットを尊重し 1:1 追従。
- **境界**: シートやスクロール端はラバーバンド(`overshoot * dim * 0.55 / (dim + 0.55 * |overshoot|)`)。ハードストップ禁止。
- **スワイプ削除(§6.4)**: 確定/復帰の判定は解放時の**速度の符号**で行い、位置だけで判定しない。

### 3.4 reduced-motion

全モーションコンポーネントに組み込みで対応する(後付けにしない):

```css
@media (prefers-reduced-motion: reduce) {
  /* スライド・スプリング → 200ms のクロスフェードへ */
}
```

motion 側は `useReducedMotion()` で分岐し、シートは transform なしの opacity フェードに落とす。カウントアップ演出も即時表示に落とす。

## 4. マテリアルと階層

### 4.1 ボトムナビ → 半透明マテリアル化

現在の不透明バー([ui.tsx:39](src/components/ui.tsx:39))を、コンテンツが下に流れ込む半透明レイヤーに置き換える:

```css
.bottom-nav {
  background: rgba(17, 17, 20, 0.72);
  backdrop-filter: blur(20px) saturate(180%);
  border-top: 1px solid rgba(255, 255, 255, 0.06);
}
```

- `PhoneShell` のスクロール領域はナビの背面まで届かせ、`padding-bottom` でコンテンツの終端だけ確保する(現行の `pb-24` を流用)。
- `prefers-reduced-transparency: reduce` では `background` を不透明の `--color-macho-base` に戻し blur を外す。

### 4.2 スクロールエッジ

ヘッダー/ナビとコンテンツの境界に 1px ボーダーではなく、**スクロール時のみ**現れるフェード(グラデーションマスク or 薄い blur)を使う。コンテンツが浮遊チェロームの下に実際に重なる場所にだけ適用する。

### 4.3 シート(新規コンポーネント、§6.1)

- モーダルな操作(記録の削除確認、エクササイズ詳細編集)は**スクリム付きシート**。背面をわずかに縮小(scale 0.97)+暗転させ「奥に押し込む」。
- 非ブロッキングな補助(サジェスト一覧など)はスクリムなしの半透明レイヤー。
- シートは blur とスケールを同時にアニメーションして「マテリアルが到着する」見せ方にする(単なる opacity フェード禁止)。

## 5. デザイントークンの再定義

### 5.1 タイポグラフィ

現状の px 直書き(`text-[34px]`, `text-[11px]` …)をやめ、rem ベースのタイプスケールをトークン化する。**ユーザーの文字サイズ設定に追従させる**(Dynamic Type 相当)。

| トークン | サイズ | line-height | letter-spacing | 用途 |
| --- | --- | --- | --- | --- |
| `display-xl` | 2.125rem (34px) | 1.0 | +0.04em ※Bebas | ページタイトル |
| `display-num` | 2rem | 1.0 | +0.02em ※Bebas | 統計数値 |
| `title` | 1rem / semibold | 1.3 | -0.01em | カード見出し |
| `body` | 0.875rem | 1.5 | 0 | 本文 |
| `caption` | 0.75rem | 1.4 | +0.01em | 補助テキスト |
| `label` | 0.6875rem | 1.3 | +0.02em | ラベル・単位 |

- Bebas Neue は縦長のディスプレイ書体なので正のトラッキングを維持(現行の `0.04em` は妥当)。Outfit の本文は `0`、小サイズのみ微プラス。
- スペーシングも rem 基準にし、文字拡大時にレイアウトごと拡大する。
- 34px 級の日本語見出し(Bebas はラテンのみ)は Outfit/システムフォントにフォールバックしている。日本語部分は `font-weight: 600` + `-0.01em` で引き締める。

### 5.2 色

現行パレットは維持し、役割トークンを追加:

- `--color-macho-danger: #ff6b6b`(現在ハードコードされている削除色を昇格)
- `--color-macho-scrim: rgba(0,0,0,0.5)`(シート用)
- マテリアル用: `--material-nav`, `--material-sheet`(§4 の rgba+blur の組)
- 部位カラー(muscle color)は現状 DB 由来のままとし、透明度の掛け方(`1f` suffix 直書き)を `color-mix()` ベースのユーティリティに統一。

### 5.3 形状

- 角丸を 3 段に整理: `--radius-s: 10px`(入力・小要素)/ `--radius-m: 14px`(カード・ボタン)/ `--radius-l: 28px`(シート・シェル)。現行の `[10px]/[12px]/[14px]` 混在を解消。
- ヒットターゲットは最小 44×44px を全対話要素で保証(現状 MiniStepper の ±ボタンは満たしている。履歴の編集ボタン 32px は要拡大)。

## 6. 画面・コンポーネント別刷新計画

### 6.1 共通コンポーネント(新規/改修)

| コンポーネント | 内容 |
| --- | --- |
| `Sheet`(新規) | 下から出るシート。1:1 ドラッグ追従、下スワイプで dismiss(速度符号で判定)、ラバーバンド、velocity handoff、スクリム+背面押し込み。入る方向と出る方向を一致させる(下から入り下へ出る)。 |
| `Toast`(新規) | 完了/エラー/Undo の3種。保存成功・削除時に表示。Undo は削除の取り消しに使う(C7, C8)。表示はナビ上端から spring で浮上。 |
| `BottomNav`(改修) | 半透明マテリアル化(§4.1)。アクティブ切替はアイコンの spring スケール+色。ラベル「ホーム」→「ダッシュボード」(C10)。中央の記録ボタンは維持(良いパターン)。 |
| `Card`(改修) | radius トークン適用のみ。視覚は現状維持。 |
| `Pill` / `ModeButton` / `RangeTabs`(改修) | 選択状態の背景を **共有レイアウトアニメーション**(motion の `layoutId`)でスライドさせる。選択が「移動」として読める=§8 ジェスチャ方向のヒント。 |
| `Stepper` / `MiniStepper`(改修) | 押下即時反応+**長押しで連続増減**(加速付き)。値変更時に数値を軽く scale パルス(damping 1.0)。ジムで手袋・片手でも連打しやすく。 |

### 6.2 ダッシュボード

- **期間タブをクライアント状態に変更**(C4)。データは全期間分を既に取得済みなので、タブ切替はフィルタのみ=即時応答。URL 同期は `router.replace`(scroll: false)で維持。
- タブ切替時、統計数値は**現在値から新値へカウントアップ/ダウン**(spring で数値補間)。reduced-motion では即時差し替え。
- 週間チャート: 偶数日だけ濃い着色(C9)を廃止し、「当日=ライム実色 / 実績日=ライム40% / 無し=10%」の意味ある符号化に変更。バーの高さは初回表示時に下から spring で伸ばす(damping 1.0)。
- 統計カードの数値は達成の中心なので `display-num` トークンで最大の視覚重量を維持。

### 6.3 記録画面(最重要フロー)

- **サジェストリストのアンカー出現**(C5): 入力欄下端を `transform-origin` に、scale 0.96→1 + フェードの spring で出現。閉じるときは同経路を逆再生(空間的一貫性)。blur 遅延タイマー(120ms)は pointer-down ハンドリングに置換して遅延を排除。
- **保存フロー**: 保存ボタン押下 → ボタン内スピナー → 成功時に Toast「保存しました」+ dashboard へ遷移。失敗はインラインではなく Toast(error)で通知し、入力は保持。
- **エクササイズ追加**: 追加された行はリスト先頭/末尾に spring で挿入(motion の layout アニメーション)。削除は行のスワイプ削除(左スワイプ→速度で確定判定)+ Undo トースト。X ボタンも残す(発見可能性)。
- 部位選択グリッド: 選択時に選択枠が spring で移動(layoutId)。
- 日付入力: ネイティブ `input[type=date]` を維持(Familiarity — iOS のホイールが最良)。

### 6.4 履歴

- フィルタ Pill をクライアントフィルタ化(C4 と同じ理由・同じ手法)。
- フィルタ切替時、カードは motion の layout アニメーションで並び替え/退出(退出は opacity+scale 0.98 のフェード、200ms 相当の spring)。
- 記録カードを**タップで展開/折りたたみ**できる検討(現状は常時全展開で長い)。展開はカード自身が伸びる spring(高さは layout アニメーション)。→ スコープ判断は実装フェーズで。
- 編集ボタンのヒットターゲットを 44px に拡大。

### 6.5 その他画面(テンプレート・設定・料金)

- 上記トークン・コンポーネント刷新の適用のみ。個別の再設計はしない(Purpose: 記録フローに投資を集中)。

## 7. アクセシビリティ対応(一括)

グローバルに 3 つのメディアクエリを導入し、コンポーネント側で分岐を持つ:

1. `prefers-reduced-motion: reduce` — スライド/スプリング → クロスフェード。パララックス・カウントアップ廃止。
2. `prefers-reduced-transparency: reduce` — ナビ・シートを不透明化、blur 除去。
3. `prefers-contrast: more` — カード・ボタンの境界線を `--color-macho-border` より明るい実線に強化。

追加: `::-webkit-scrollbar { width: 0 }` の全面非表示([globals.css:54](app/globals.css:54))はデスクトップで位置手掛かりを失うため、ポインタ環境(`@media (pointer: fine)`)では細いスクロールバーを復活させる。

## 8. 実装フェーズ(提案)

依存関係順。各フェーズは独立してリリース可能。

| フェーズ | 内容 | 対象 |
| --- | --- | --- |
| **P1: 基盤** | デザイントークン刷新(§5)+ a11y メディアクエリ(§7)+ motion 導入とスプリング定数モジュール(§3.2) | globals.css, ui.tsx, 新 `src/lib/motion.ts` |

| **P3: 即時応答** | ダッシュボード期間タブ・履歴フィルタのクライアント化+共有レイアウト選択(C4) | dashboard, history |

## 9. 非対象(やらないこと)

- ライトテーマ対応(ブランドはダーク固定。`color-scheme: dark` 維持)
- 配色・ロゴ・書体ファミリーの変更
- ハプティクス/サウンド(Web の Vibration API は iOS Safari 非対応のため見送り)
- デスクトップ専用レイアウトの新設(PhoneShell の中央寄せを維持)
- 情報設計(画面構成・ナビ構造)の変更 — 3タブ+中央記録ボタンは維持
