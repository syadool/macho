# 要件定義書 — デザイン刷新（Apple Design 原則の適用）

対象: MACHO（筋トレ記録 Web アプリ / Next.js 15 + React 19 + Tailwind v4 + Supabase）
上位文書: [SPECIFICATION_DESIGN_REDESIGN.md](./SPECIFICATION_DESIGN_REDESIGN.md)（刷新計画。課題 C1–C10 と方向性はここで定義済み）
作成: Fable（要件定義フェーズ）/ 2026-07-11

---

## 1. 目的

「ジムで片手・30秒で記録できる」体験を損なわずに、Apple のデザイン原則（レスポンス即時性・中断可能なスプリングモーション・マテリアル階層・アクセシビリティ・寛容な操作）をアプリ全体へ適用する。飾りのアニメーションは追加せず、記録フローの速さ・確実さ・達成感に投資する。

## 2. スコープ

## 3. 制約・前提

- **既存ブランドの維持**: ダーク基調 + ライム `#d4ff00` + Bebas Neue の数字。配色・書体は変更しない。
- **依存追加は `motion` のみ**。他の UI ライブラリ・アニメーションライブラリは追加しない。
- **サーバーコンポーネント構成の尊重**: dashboard / history は現在 RSC。クライアント化はフィルタ UI とリスト描画に必要な最小範囲に留め、データ取得（`getAllWorkouts` / `getWorkouts` / `getMasterData`）はサーバー側のまま初期 props で渡す。
- **URL 互換**: `/dashboard?range=` と `/history?muscle=` の URL は引き続き有効（直リンク・リロードで同じ状態を復元）。
- **iOS Safari が主要ターゲット**: backdrop-filter は `-webkit-` を考慮。`svh` / セーフエリア対応の既存挙動を壊さない。
- **既存機能の回帰禁止**: 記録の保存・編集・削除、テンプレート、課金導線の動作を変えない（見た目とフィードバックのみ変更）。

## 4. 受け入れ基準

1. `npm run lint` と `npm run build` が成功する。
2. ダッシュボードの期間タブ・履歴のフィルタが、サーバー往復なしで即時に切り替わり、URL も同期される。リロード/直リンクで同じ状態が再現される。
3. ボトムナビが半透明+blur になり、コンテンツがその背面にスクロールして見える。`prefers-reduced-transparency: reduce` で不透明に戻る。
4. `prefers-reduced-motion: reduce` 環境で、スライド・スプリングがフェード or 即時反映に落ちる（motion コンポーネントは `useReducedMotion` 分岐を持つ）。
5. 主要テキストが rem ベースになり、ブラウザの文字サイズ変更（例: 125%）に追従して拡大する。
6. 記録フォームでエクササイズを削除すると Undo トーストが出て、タップで復元できる。Undo 期限（5 秒目安）経過後に確定する。
7. 保存成功時にトースト表示があり、失敗時は入力が保持されたままエラートーストが出る。
8. 履歴の編集ボタンほか対話要素の当たり判定が 44×44px 以上。
9. チャートの着色が「当日 / 実績あり / 無し」の 3 値符号化になっている。

## 5. 検証方法

- `npm run lint` / `npm run build`（既存テストスイートは存在しないため、この 2 つを機械検証とする）
- 上記受け入れ基準の目視確認項目は、実装報告に「確認手順」として記載する。
