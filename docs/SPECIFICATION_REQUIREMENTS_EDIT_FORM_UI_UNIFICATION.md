# 要件定義書: 編集画面と新規作成画面のUI統一

- 作成日: 2026-07-13
- 作成者: Fable(メインセッション)
- ステータス: 確定(sol 最終チェック GO 判定済み)

## 1. 背景

ワークアウトの新規作成画面(`/record`)と編集画面(`/history/[id]/edit`)は同じ概念(種目タイプ切替、部位選択、エクササイズ名入力+サジェスト、セット編集、有酸素の時間入力)を扱うが、UIコンポーネントがそれぞれのファイルにローカル定義されており、見た目・挙動が乖離している。

ユーザーからの指摘: 「編集画面と新規作成画面でUIが違うのが気になるので、新規作成画面側に統一してください」

### 現状の乖離点(統一前)

| 要素 | 新規作成画面(基準) | 編集画面(統一前) |
|------|------|------|
| モード切替(筋トレ/有酸素) | h-12、text-sm、rounded-macho-m、bg-macho-card、motion whileTap | h-10、text-xs、rounded-[12px]、bg-macho-surface、モーションなし |
| 部位選択 | 3列グリッド、大タイル(p-3.5)、部位色付き22px部位名 + 英語名サブタイトル、motion whileTap、「部位を選択」ラベルあり | 3列グリッド、小ボタン(px-2 py-2.5、text-xs)、色なし、ラベルなし |
| 有酸素の時間ステッパー | Cardベース、数値26px、gap-2 | 素のdiv、数値24px、gap-1.5 |
| エクササイズ名サジェスト | 「前回: ○kg×○回×○セット」等のサブタイトル表示あり | 種目名のみ(サブタイトルなし) |
| エクササイズ名入力 | プレースホルダあり(ベンチプレス/ランニング) | プレースホルダなし |

## 2. 目的

1. 編集画面のUIを新規作成画面のスタイルに統一し、画面間の一貫性を確保する。
2. 重複コンポーネント定義を共有モジュールに抽出し、今後の乖離を構造的に防止する。

## 3. スコープ

### 対象

- `app/record/record-form.tsx`(新規作成画面): 見た目は不変のまま、ローカル定義を共有コンポーネントのインポートに置換。
- `app/history/[id]/edit/edit-workout-form.tsx`(編集画面): 共有コンポーネント(新規作成画面スタイル)への置換。
- 共有コンポーネントの新設: `src/components/workout-form.tsx`
  - `ModeButton` — 筋トレ/有酸素トグル(新規作成画面スタイル)
  - `MuscleGroupGrid` — 部位選択タイル(色付き部位名 + 英語名)
  - `ExerciseSuggestionList` — サジェストリスト(「前回: ...」サブタイトル付き)
  - `DurationStepper` — 有酸素の時間ステッパー(Cardベース)
- 付随リファクタ(技術選定で確定。見た目の変化なし):
  - `app/globals.css` に `--color-macho-border-hover: #555555` トークンを追加し、既存の `hover:border-[#555]` 4箇所(workout-form.tsx / ui.tsx の Pill / templates/page.tsx / onboarding-form.tsx)を機械置換
  - `src/components/stepper.tsx` の `PressAndHoldStepperButton` に reduced-motion 対応を追加
  - `ModeButton` に `aria-pressed={active}` を付与

### 対象外

- 編集画面の操作モデル(種目ごとのカードで全種目を同時編集できる構造)は変更しない。新規作成画面の「1種目ずつ入力→サマリーカードに追加」モデルへの変更は行わない(既存種目の編集ができなくなる退行のため)。
- `SetRowsEditor`(`src/components/workout-sets.tsx`)は既に共有済みのため変更しない。
- `src/components/ui.tsx` の BottomNav 差分(flex→grid)は本テーマと無関係の既存変更であり触れない。
- テンプレート画面・ダッシュボード等、他画面のUIは対象外。

## 4. 機能要件

- FR-1: 編集画面のモード切替・部位選択・時間ステッパー・サジェストの見た目が新規作成画面と同一であること。
- FR-2: 新規作成画面の見た目・挙動は統一前と同一であること(リグレッションなし)。
- FR-3: 編集画面の既存機能が維持されること:
  - サジェスト適用(種目名・部位・前回セット/時間の反映)
  - セット追加/削除と削除Undoトースト
  - 種目追加/削除と削除Undoトースト
  - 部位変更時の `muscle_sub_group_ids` リセット
  - 保存(updateWorkout)フロー
- FR-4: サジェストリストに「前回: ...」サブタイトルが両画面で表示されること。

## 5. 非機能要件

- NFR-1: `npx tsc --noEmit` および lint がパスすること。
- NFR-2: アクセシビリティ属性(aria-label、min-h-11/min-w-11 のタップターゲット)を既存水準から劣化させない。
- NFR-3: `useReducedMotion` によるモーション低減対応を共有コンポーネント側で維持する。
- NFR-4: 共有コンポーネントは MACHO デザイントークン(macho-lime、macho-border、rounded-macho-m 等)を使用する。色リテラルの直書きは避け、必要な場合は `app/globals.css` の `@theme inline` へのトークン追加を許可する(例: `macho-border-hover`)。

## 6. 受け入れ基準

1. 編集画面と新規作成画面で、対象4コンポーネントの見た目が一致する。
2. 両画面のフォーム操作(入力→保存)が統一前と同様に完了できる。
3. 型チェック・lintがエラーなしでパスする。
4. `app/record/record-form.tsx` と `app/history/[id]/edit/edit-workout-form.tsx` に対象コンポーネントのローカル定義が残っていない。
